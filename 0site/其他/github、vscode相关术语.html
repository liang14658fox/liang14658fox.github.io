这些术语的解释旨在提供一个快速参考，以帮助理解GitHub和VSCode的基本操作和功能。

GitHub 相关术语 ​

Archive - 归档，GitHub上的一个功能，允许你将项目存档，使其只读，不再主动维护。

Atom - Atom，GitHub开发的一个开源文本编辑器，支持多种编程语言。

Base Branch - 基础分支，合并拉取请求时目标分支，通常是主分支如main或master。

Blame - 责任追踪，GitHub的一个功能，用于查看文件的每一行是谁最后修改的。

Clone - 克隆，复制GitHub仓库到本地计算机。

Code Review - 代码审查，审查团队成员的代码更改，以保证代码质量和一致性。

Collaborator - 协作者，被仓库所有者授权可以直接推送到仓库的GitHub用户。

Commit Hash - 提交哈希，唯一标识一个提交的40位十六进制数字。

Contribution - 贡献，指任何形式的工作，通过提交、问题讨论或拉取请求等形式贡献给项目。

Dashboard - 仪表盘，GitHub用户的个人主页，展示了活动、仓库、正在进行的工作等信息。

Diff - 差异，显示文件更改前后的差别。

Forking - 分叉，复制别人的GitHub仓库到你的GitHub账户下，以便独立开发。

Gist - Gist，GitHub提供的一个服务，用于分享代码片段或其他小段文本。

Issue Label - 问题标签，用于分类和组织仓库中的问题。

Markdown - Markdown，一种轻量级标记语言，GitHub支持用其编写文档。

Merge Conflict - 合并冲突，当两个分支在同一文件的同一部分都进行了更改时发生。

Milestone - 里程碑，用于追踪项目进度的特定目标或时间点。

Notification - 通知，GitHub上的更新或活动提醒。

OAuth - OAuth，一种开放标准，允许用户授权第三方应用访问他们在GitHub上的信息，而无需直接交出密码。

Organization - 组织，GitHub上的一个账户类型，用于群组协作。

Owner - 拥有者，仓库或组织的创建者或主要负责人。

Page - 页面，GitHub Pages是一项服务，允许用户托管静态网站。

Private Repository - 私有仓库，只有授权用户可以访问的仓库。

Pull Request Review - 拉取请求审查，审查并可能提出更改建议的过程。

Push - 推送，将本地的代码更改上传到GitHub仓库。

README - README文件，通常是一个仓库的介绍文件，解释项目是什么、如何运行等。

Release - 发布，标记特定的项目状态（如版本）并提供下载。

Repository Settings - 仓库设置，用于配置仓库选项和权限的地方。

SSH Key - SSH密钥，一种安全协议，用于在本地计算机和GitHub之间建立安全连接。

Staging Area - 暂存区，Git中的一个概念，暂存你想要提交的更改。

Star -星标，用户可以给他们喜欢的仓库加星标，以便于找到或表示支持。

Status Checks - 状态检查，自动测试和其他检查，用于在合并拉取请求前验证更改。

Tag - 标签，用于标记特定的提交，通常用于版本发布。

Team - 团队，组织内部的一组用户，可以分配特定的权限和任务。

Webhook - Webhook，一种方法，允许外部服务在GitHub仓库中发生特定事件时接收通知。

Workflow - 工作流，GitHub Actions中定义的一系列自动化步骤，用于CI/CD等任务。

VSCode 相关术语 ​

Breadcrumbs - 面包屑，VSCode中的导航辅助功能，显示当前文件的路径。

Code Folding - 代码折叠，允许用户隐藏代码块的功能，以便于集中注意力在特定部分。

Debug Console - 调试控制台，用于显示调试信息和与调试会话交互。

Extension - 扩展，可以添加到VSCode以增强其功能的插件或工具。

IntelliSense - 智能感知，VSCode的代码自动完成和信息提示功能。

Key Binding - 键绑定，自定义键盘快捷键以执行VSCode命令。

Linting - 代码检查，自动检查代码错误和风格问题的工具。

Minimap - 缩略图，VSCode中显示文件概览的小地图。

Output Panel - 输出面板，显示各种运行信息和日志的地方。

Peek Definition - 查看定义，允许在不离开当前位置的情况下查看代码定义的功能。

Quick Fix - 快速修复，提供针对代码问题的即时修复建议。

Refactor - 重构，修改代码结构而不改变其外在行为的过程。

Snippet - 代码片段，预定义的小块代码，可以快速插入到文件中。

Source Control - 源代码控制，VSCode内置的版本控制集成，支持Git等。

Terminal - 终端，VSCode内置的命令行工具，允许用户直接在编辑器内执行命令。

Theme - 主题，改变VSCode颜色和样式的设置。

User Settings - 用户设置，自定义VSCode行为和外观的配置。

Workspace - 工作区，一组相关项目的集合，VSCode可以同时打开和管理。
